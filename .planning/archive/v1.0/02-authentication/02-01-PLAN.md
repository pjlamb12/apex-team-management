Rewrite `apps/api/src/main.ts` to add:
1. Global `ValidationPipe` with `{ whitelist: true }` — strips unknown properties from request bodies so DTO class-validator decorators enforce shape.
2. CORS enabled for `http://localhost:4200` (Angular dev server origin).

Target content:

```typescript
import 'reflect-metadata';
import { NestFactory } from '@nestjs/core';
import { ValidationPipe } from '@nestjs/common';
import { AppModule } from './app/app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.useGlobalPipes(new ValidationPipe({ whitelist: true }));

  app.enableCors({
    origin: 'http://localhost:4200',
    credentials: true,
  });

  const port = process.env['PORT'] ?? 3000;
  await app.listen(port);
}
bootstrap();
```

The `reflect-metadata` import MUST remain first — it enables TypeORM and NestJS decorator metadata.

**Step B — Update `apps/api/.env`:** Append the same two lines with a local dev value. Generate a random secret using Node:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```
Use the output as the JWT_SECRET value. The `.env` file is gitignored — safe to put real entropy here.

**Step C — Create `apps/frontend/proxy.conf.json`:**
```json
{
  "/api": {
    "target": "http://localhost:3000",
    "secure": false,
    "changeOrigin": true,
    "logLevel": "info"
  }
}
```
This routes all `/api/*` requests from the Angular dev server to the NestJS API. The auth service calls `/api/auth/login`, `/api/auth/signup`, etc. — the proxy strips the host so NestJS receives `/api/auth/login` relative to its root.

NestJS routes are prefixed with `api` via `app.setGlobalPrefix('api')` — add this to `main.ts` AFTER the ValidationPipe line:
```typescript
app.setGlobalPrefix('api');
```

Update `apps/api/src/main.ts` to include `app.setGlobalPrefix('api')` before `app.listen()`. The AuthController uses `@Controller('auth')` so endpoints become `/api/auth/signup`, `/api/auth/login`, etc.

**Step D — Update `apps/frontend/project.json` serve target:** Add `proxyConfig` option to the serve target's development configuration:
