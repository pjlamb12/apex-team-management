# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Scope Note

Phase 1 has **no user-facing screens or interactive flows**. The "UI" deliverable in this phase
is the theme infrastructure that every subsequent phase consumes:

- Tailwind CSS configured with "Athletic Professional" dark mode tokens
- Ionic Framework installed and CSS variables overridden under `.dark`
- `ThemeService` in `libs/client/ui/theme` — canonical theme authority for the whole app
- `styles.scss` entry point with Tailwind directives and Ionic imports

This contract defines the design tokens (color, spacing, typography) that the executor wires
into `tailwind.config.js` and the Ionic CSS variable overrides. Phases 2–6 inherit these tokens
and must not redefine them.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, list item gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, form field groups |
| xl | 32px | Layout gaps, major card margins |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing, hero areas |

Exceptions:
- Touch targets (buttons, tap areas): minimum 44px height — Ionic handles this via `ion-button` defaults; do not override below 44px
- Ion list items: 48px height minimum for comfortable thumb reach

**Source:** Default 8-point scale. Touch target exception per REQUIREMENTS.md INFR-01 (mobile-first).

## Color

### Design Intent

"Athletic Professional" — dark mode primary, clean dashboard-style, high-contrast. Reads like a
coaching tool, not a consumer app. Dark surfaces with high-contrast text and a single vibrant
accent for action elements.

### Color Tokens

| Role | Dark Mode Value | Light Mode Value | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) | `#111827` (gray-900) | `#F9FAFB` (gray-50) | Page background, app shell |
| Secondary (30%) | `#1F2937` (gray-800) | `#FFFFFF` (white) | Cards, nav bar, side panels, ion-list items |
| Accent (10%) | `#10B981` (emerald-500) | `#059669` (emerald-600) | Primary action buttons, active tab indicator, ThemeService toggle button, focus rings |
| Destructive | `#EF4444` (red-500) | `#DC2626` (red-600) | Destructive action buttons only |

**Accent reserved for (explicit list):**
1. Primary CTA button background (phases 2–6 — not used in Phase 1)
2. Active tab indicator (bottom navigation)
3. ThemeService dark/light toggle control
4. Focus ring outline color (accessibility)

Accent is NOT used for: general borders, hover states, informational icons, or secondary buttons.

**Text colors:**
- Primary text (dark mode): `#F9FAFB` (gray-50)
- Secondary text (dark mode): `#9CA3AF` (gray-400)
- Primary text (light mode): `#111827` (gray-900)
- Secondary text (light mode): `#6B7280` (gray-500)

### Dark Mode Strategy

- **Tailwind:** `darkMode: 'class'` in `tailwind.config.js`. The `.dark` class is applied to `<html>` by `ThemeService`.
- **Ionic:** CSS custom property overrides scoped under `.dark` in `styles.scss`. Key overrides:
  - `--ion-background-color: #111827`
  - `--ion-surface-color: #1F2937`
  - `--ion-text-color: #F9FAFB`
  - `--ion-color-primary: #10B981`
  - `--ion-color-danger: #EF4444`
- **Default:** OS preference (`prefers-color-scheme`) on first load. Manual toggle persists to `localStorage`.

**Source:** CONTEXT.md D-07, D-08, D-09; REQUIREMENTS.md INFR-02; TRD Section 5.

## Tailwind Configuration Contract

The executor must wire the following into `tailwind.config.js` (or `tailwind.config.ts`):

```js
// Extend theme — do not override Tailwind defaults entirely
theme: {
  extend: {
    fontFamily: {
      sans: ['Inter', '-apple-system', 'BlinkMacSystemFont', 'Segoe UI', 'sans-serif'],
    },
    colors: {
      'ap-bg': { dark: '#111827', light: '#F9FAFB' },
      'ap-surface': { dark: '#1F2937', light: '#FFFFFF' },
      'ap-accent': { DEFAULT: '#10B981', dark: '#10B981', light: '#059669' },
      'ap-danger': { DEFAULT: '#EF4444', dark: '#EF4444', light: '#DC2626' },
    },
    // Standard spacing already present in Tailwind — no overrides needed
  }
}
```

`darkMode: 'class'` must be declared at the top level of the config.

**Content paths must include:**
- `apps/frontend/src/**/*.{html,ts}`
- `libs/**/*.{html,ts}` (for when libs are created in later phases)

## ThemeService Contract

`libs/client/ui/theme` must export:

| Export | Type | Responsibility |
|--------|------|----------------|
| `ThemeService` | Angular service (injectable in root) | Manages `isDark` signal; applies/removes `.dark` on `<html>` |
| `isDark` | `Signal<boolean>` | Current theme state — readable by any component |
| `toggle()` | method | Flips theme; persists to `localStorage` under key `'apex-theme'` |
| `init()` | method | Called at app startup (in `app.config.ts`); reads `localStorage` then falls back to `prefers-color-scheme` |

No UI component is shipped from this lib in Phase 1. The toggle button component is scaffolded
in Phase 2 or 3 when a nav bar exists to place it in.

**Source:** CONTEXT.md D-07, D-08.
