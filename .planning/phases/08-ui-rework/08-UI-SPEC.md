## Root Cause Analysis

The following Ionic iOS dark palette defaults are the primary source of pure-black backgrounds:

```
.ion-palette-dark.ios {
  --ion-background-color: #000000;   /* PAGE background — pure black */
  --ion-item-background: #000000;    /* ion-item rows — pure black */
  --ion-card-background: #1c1c1d;    /* cards — near-black */
}
```

Additionally, `console-wrapper.scss` hard-codes `--background: #000000` on `ion-toolbar`
and `background: #111111` on the scoreboard container.

`event-log.scss` falls back to `#ffffff` (white) in its host block, causing a white panel
inside the dark game console when no Ionic variable is resolved.

`libs/client/ui/theme/src/lib/theme/theme.css` is empty — no global token overrides exist yet.

Fix strategy: Override `--ion-background-color` and related variables in `theme.css` (applied
globally when `.ion-palette-dark` is active) and then clean up per-component hardcodes.

## Color Contract

### Dark Palette — "Athletic Professional" Slate

No pure blacks. All surfaces use dark slate/charcoal from the blue-gray spectrum.

| Token Name | Hex | Luminance | Usage |
|------------|-----|-----------|-------|
| `--ap-bg` | `#0f172a` | 6% | Page background (replaces `#000000`) |
| `--ap-surface` | `#1e293b` | 12% | Cards, modals, popovers, bench cards |
| `--ap-surface-raised` | `#263044` | 16% | Elevated cards, toolbar, console top bar |
| `--ap-border` | `rgba(255,255,255,0.08)` | — | Dividers, card borders, list separators |
| `--ap-text-primary` | `#f1f5f9` | 94% | Headings, labels, primary copy |
| `--ap-text-secondary` | `#94a3b8` | 60% | Subtitles, metadata, timestamps |
| `--ap-text-muted` | `#475569` | 33% | Placeholder copy, hints (do NOT use on body text — fails WCAG AA) |
| `--ap-accent` | `#3b82f6` | — | Reserved — see Accent section below |
| `--ap-accent-glow` | `rgba(59,130,246,0.20)` | — | Box-shadow accent bloom only |
| `--ap-success` | `#22c55e` | — | Confirmed sub, goal confirmation |
| `--ap-warning` | `#eab308` | — | Yellow card, pause state |
| `--ap-danger` | `#ef4444` | — | Red card, destructive actions |

### 60 / 30 / 10 Distribution

| Role | Token | Surfaces |
|------|-------|---------|
| 60% Dominant | `--ap-bg` (`#0f172a`) | `ion-content` page background, `ion-tab-bar` backing |
| 30% Secondary | `--ap-surface` (`#1e293b`) | `ion-card`, `ion-list`, `ion-item`, `ion-modal`, bench cards, event-log panel |
| 10% Accent | `--ap-accent` (`#3b82f6`) | See below |

### Accent Reserved For (10% — exclusive list)

- Primary `ion-button` fill
- Active `ion-segment-button` indicator
- `ion-badge color="primary"`
- Selected bench card highlight border
- Scoreboard box border glow
- `ion-fab-button` background
- Link text (`<a>` tags in auth pages)

Accent is NOT used for: hover states, general backgrounds, text color on body copy, inactive icons.

| `#94a3b8` on `#0f172a` | 7.1:1 | PASS |
| `#94a3b8` on `#1e293b` | 5.5:1 | PASS |

## Ionic CSS Variable Overrides

These go in `libs/client/ui/theme/src/lib/theme/theme.css`, scoped to `.ion-palette-dark`.
They replace the Ionic iOS `#000000` defaults with the AP slate palette.

```css
/* Athletic Professional — Ionic dark palette overrides */
/* Applied when ThemeService sets .ion-palette-dark on <html> */

.ion-palette-dark {
  /* Page and content background — replaces iOS #000000 */
  --ion-background-color: #0f172a;
  --ion-background-color-rgb: 15, 23, 42;

  /* Item/row background — replaces iOS #000000 */
  --ion-item-background: #0f172a;

  /* Card background — replaces iOS #1c1c1d */
  --ion-card-background: #1e293b;

  /* Toolbar background — replaces console-wrapper.scss hardcode #000000 */
  --ion-toolbar-background: #0f172a;

  /* Tab bar */
  --ion-tab-bar-background: #0f172a;
  --ion-tab-bar-border-color: rgba(255, 255, 255, 0.08);

  /* Text */
  --ion-text-color: #f1f5f9;
  --ion-text-color-rgb: 241, 245, 249;

  /* Color steps — used by components that reference step-50, step-100, etc. */
  /* Step 50: card/surface color */
  --ion-color-step-50: #1e293b;
  --ion-color-step-100: #263044;
  --ion-color-step-150: #2d3a52;
  --ion-color-step-200: #374151;
}
```

**Scope note:** The `.ios` qualifier in Ionic's dark.class.css is what makes iOS use `#000000`.
Overriding without the `.ios` qualifier at a higher specificity (same `.ion-palette-dark` level
but declared after the Ionic import in styles.scss) will win. Confirm styles.scss loads
`theme.css` AFTER the Ionic dark palette import.

## Tailwind CSS v4 Custom Tokens

Add to `apps/frontend/src/styles.scss` inside an `@layer base` or as `@theme` block
(Tailwind v4 syntax — no config file needed):

```css
@theme {
  --color-ap-bg: #0f172a;
  --color-ap-surface: #1e293b;
  --color-ap-surface-raised: #263044;
  --color-ap-accent: #3b82f6;
  --color-ap-success: #22c55e;
  --color-ap-warning: #eab308;
  --color-ap-danger: #ef4444;
  --color-ap-text: #f1f5f9;
  --color-ap-muted: #94a3b8;
}
```

This enables Tailwind utilities like `bg-ap-bg`, `text-ap-muted`, `border-ap-surface` across
all templates. The `dark:` prefix is not needed because the app renders in dark mode globally —
`ThemeService` always applies `.dark` on load for new sessions.

## Spacing Scale

All values are multiples of 4px. This matches the existing usage found across templates
(p-4, gap-2, mb-4, py-2, px-1.5, etc.).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding, small badge padding |
| sm | 8px | Row inner padding, tight list gaps |
| md | 16px | Default `ion-content` padding, card horizontal padding |
| lg | 24px | Section separation within a page |
| xl | 32px | Major section breaks (e.g. empty state illustration gap) |
| 2xl | 48px | Page-level vertical breathing room |
| 3xl | 64px | Not used in current phase |

**Exceptions:**
- Touch targets (player circles, bench cards, FAB): minimum 44px × 44px per iOS HIG
- `ion-toolbar` height in game console: 80px (existing — keep as-is, it's intentional for scoreboard legibility)
- `ion-select` minimum height: 44px (existing in lineup-editor.scss — keep)

## Styles.scss Import Order Requirement

The current `apps/frontend/src/styles.scss` MUST maintain this load order:

1. `@import "tailwindcss"` — first
2. `@custom-variant dark` — after tailwind
3. `@source` declarations — after tailwind
4. `@import "@ionic/angular/css/core.css"` + siblings
5. `@import "@ionic/angular/css/palettes/dark.class.css"` — Ionic dark vars
6. **`@import` or `@source` for `theme.css`** — MUST be last to win specificity

If `theme.css` is not currently imported in `styles.scss`, add:
```css
@import "../../../../libs/client/ui/theme/src/lib/theme/theme.css";
```

## Copywriting Contract

This is a rework phase — no new UX flows. Existing copy is generally correct.
These are the standards for any copy touched during cleanup:

| Element | Copy Standard |
|---------|--------------|
| Primary CTA (forms) | Verb + noun: "Create Team", "Schedule Game", "Save Lineup", "Sign In" |
| Loading state | Verb + "...": "Signing in...", "Saving...", "Loading..." |
| Empty state heading | "No [noun] yet" — e.g. "No teams yet", "No events logged yet." |
| Empty state body | "Create your first [noun] to get started" |
| Error state | State the problem, not the code: "Couldn't load teams. Check your connection and retry." |
| Destructive confirm | Alert title: "Delete [Team Name]?" Body: "This can't be undone." Buttons: "Cancel" / "Delete" |

**Existing copy that violates this standard and must be updated:**
- `team-dashboard.html` empty roster: "No players on this roster yet." — acceptable, keep.
- `team-dashboard.html` empty games: "No games scheduled yet." — acceptable, keep.
- `lineup-editor.html` error state: Shows raw error string `{{ errorMessage() }}` — add a friendly prefix if the error is API-sourced.
