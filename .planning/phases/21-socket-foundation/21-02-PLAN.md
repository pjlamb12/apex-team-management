# Plan 21-02: Socket.io Frontend Service & Room Logic

Implement the frontend service to manage Socket.io connections and room memberships.

## Tasks

### 1. Install Frontend Dependencies
- Install `socket.io-client`.

### 2. Create SocketService
- Implement a service that establishes a connection to the backend using the user's JWT.
- Handle reconnection logic and connection status reporting (e.g., via a signal).
- Implement methods to join and leave rooms based on the current context (team dashboard or game console).

### 3. Integrate with App Configuration
- Register `SocketService` as a provider in `app.config.ts`.

### 4. Update Navigation/Components
- Ensure that when a user navigates to a team or event, the `SocketService` joins the appropriate room.

## Verification Strategy

### Automated Tests
- Create unit tests for `SocketService` to verify that it correctly uses the JWT for authentication.
- Verify that the service attempts to reconnect on failure.

### Manual Verification
- Open the browser console and verify that the socket connection is established.
- Verify that the connection is closed when the user logs out.
- Use network monitoring to see the join/leave room messages.
