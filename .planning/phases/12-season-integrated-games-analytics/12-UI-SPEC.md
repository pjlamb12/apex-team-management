## Spacing Scale

Declared values (multiples of 4 only). Tailwind v4 utility classes are the primary delivery mechanism; Ionic's `ion-padding` / `ion-margin` helpers are acceptable for Ionic-owned Shadow DOM regions.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap, inline padding between icon and label (e.g., `style="margin-inline-end: 4px"`) |
| sm | 8px | Compact row gap, picker margin-bottom before segment |
| md | 16px | Default element spacing, `ion-card-content` padding, form field vertical gaps (`class="mt-3"`) |
| lg | 24px | Card header padding (`px-6`, `pt-6`), section separation |
| xl | 32px | Page-level top padding (`pt-4` + card spacing) |
| 2xl | 48px | Stat card grid vertical rhythm on season detail |
| 3xl | 64px | Not used in this phase |

Exceptions:
- `margin-inline-end: 12px` on leading `ion-icon` in list rows (established pattern — preserve, do not normalize to 8px or 16px).
- `ion-fab` bottom offset: `margin-bottom: 60px` (tab bar clearance — established pattern, preserve).
- Touch targets for score increment/decrement controls (if used): minimum 44px hit area.

## Typography

Ionic typography CSS is loaded globally. All sizes below refer to the rendered CSS value on mobile (MD/iOS platform default base is 16px).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / list label | 16px | 400 (regular) | 1.5 |
| Supporting text / `ion-note`, hints | 12px | 400 (regular) | 1.4 |
| Card title / section heading | 20px | 700 (bold) | 1.2 |
| Stat display value (W/L/D/GF/GA/GD) | 28px | 700 (bold) | 1.0 |

**Source:** `ion-card-title class="text-2xl font-bold"` (24px) is existing precedent; stat display uses 28px to give the numbers visual hierarchy above their labels.

Tailwind class mappings:
- 12px = `text-xs` (0.75rem)
- 16px = `text-base` (1rem)
- 20px = `text-xl` (1.25rem)
- 28px = `text-3xl` (1.875rem — use `text-[28px]` if exact value is needed)

## Component Inventory

New and modified components for this phase.

### 1. Season Picker (Schedule Tab)

**Placement:** Above the Upcoming/Past `ion-segment` on the Schedule component.

**Component:** `ion-select` with `ion-select-option` per season. Wrapped in `ion-item` to match existing form field style.

**Behavior:**
- Loads all team seasons via `SeasonsService.findAllForTeam()` on Schedule init.
- Defaults to the active season (pre-select on load).
- Emits `ionChange` → triggers `loadEvents()` with selected `seasonId`.
- Active season option displays `"${season.name} (Active)"` suffix as the option label text — no extra icon needed.

**Layout:**
```
[ion-item full-width]
  [ion-select label="Season" labelPlacement="floating"]
    [ion-select-option value="{season.id}"] {season.name} [(Active)]
```

**States:**
- Loading seasons: `ion-spinner name="crescent"` centered in place of the picker row.
- No seasons (edge case): Show static label "No seasons yet — create a season to get started." in muted text; picker hidden.

---

### 2. Score Entry Fields (Edit Event Form — Games Only)

**Placement:** Conditionally rendered below `Uniform Color` field and above `Duration (minutes)`, visible only when `event().type === 'game'`.

**Fields:**
- `goalsFor` — `ion-input type="number"`, label "Goals For (Our Team)", `labelPlacement="floating"`, `min="0"`, `step="1"`.
- `goalsAgainst` — `ion-input type="number"`, label "Goals Against (Opponent)", `labelPlacement="floating"`, `min="0"`, `step="1"`.

**Layout:** Two separate `ion-item` rows (same pattern as existing inputs), wrapped in `ngx-control-errors-display`. Wrapped in an `@if (event()?.type === 'game')` block.

**Score Confirmation Flow (D-08):**

When a game's `status` is `completed` and the edit event form loads:
- If `goalsFor` is null, pre-populate it with the count of logged GOAL `GameEventEntity` records (returned by the backend).
- `goalsAgainst` remains empty/null until manually entered.
- Display an informational `ion-note` below `goalsFor`: `"Pre-filled from logged goals — confirm before saving."` Text color: `--color-ap-muted` (`#94a3b8`).

**Validation:**
- Both fields accept `null` (nullable). No `Validators.required` applied.
- `Validators.min(0)` applied to prevent negative scores.
- If one field is filled and the other is not, no block — partial scores are valid.

---

### 3. Season Stats Section (Season Detail Screen)

**Placement:** New read-only section added to the Season Detail screen (`season-detail.html`), rendered BELOW the edit form, separated by a section heading.

**Section Heading:** `<h2 class="text-xl font-bold px-4 mt-6 mb-3">Season Stats</h2>`

**Layout: Stat Card Grid**

Use a 3-column CSS grid of compact stat cards inside an `ion-card`.

```
[ion-card class="mt-4"]
  [ion-card-header]
    [ion-card-title class="text-base font-bold"] Performance
  [ion-card-content]
    [div class="grid grid-cols-3 gap-3"]
      [div.stat-card] W
      [div.stat-card] L
      [div.stat-card] D
      [div.stat-card] GF
      [div.stat-card] GA
      [div.stat-card] GD
```

**Stat Card Anatomy (each cell):**
```
[div class="flex flex-col items-center bg-ap-surface-raised rounded-lg p-3 border-l-[3px] border-ap-accent"]
  [span class="text-[28px] font-bold leading-none text-ap-text"] {value}
  [span class="text-xs text-ap-muted mt-1"] {label}
```

Stat labels (use abbreviations to fit 3-column layout):
| Key | Label |
|-----|-------|
| wins | W |
| losses | L |
| draws | D |
| goalsFor | GF |
| goalsAgainst | GA |
| goalDifference | GD |

Goal Difference display: Show `+{n}` for positive, `-{n}` for negative, `0` for zero. Apply `text-ap-success` (`#22c55e`) for positive GD, `text-ap-danger` (`#ef4444`) for negative GD, `text-ap-muted` for zero.

**Stats Loading State:** Replace the grid with a centered `ion-spinner name="crescent"` while `GET /teams/:teamId/seasons/:seasonId/stats` is in flight.

**Stats Empty State:** When no completed games exist for the season (all stats are 0 / server returns zeros), show the grid with all zeros (do not hide the grid). Below the grid, render:

```
[p class="text-xs text-center text-ap-muted mt-2 pb-2"]
  Complete games and enter scores to see stats here.
```

**Stats Error State:** If the stats endpoint fails, replace the grid with:
```
[ion-text color="danger"]
  [p class="text-sm text-center py-4"] Could not load stats. Pull to refresh.
```

## Interaction States

### Season Picker

| State | Visual |
|-------|--------|
| Default | Active season pre-selected, label "Season", floating label style |
| Tap to open | Ionic native select popover/action sheet (platform default) |
| Active option | Name + " (Active)" suffix, no extra badge needed inside popover |
| Changed | Spinner replaces event list; list reloads for selected season |
| Loading seasons | Spinner in place of picker row |
| No seasons | Static muted text: "No seasons yet — create a season to get started." Picker hidden. |

### Score Entry

| State | Visual |
|-------|--------|
| Not a game | Fields hidden (conditional render) |
| Game, scores null | Fields empty, no pre-fill note |
| Game, status=completed, goalsFor pre-filled | goalsFor pre-populated with GOAL event count; info note visible below field |
| Saving | "Saving..." button label, `[disabled]="isSaving()"` on submit button (existing pattern) |
| Validation error | `ngx-control-errors-display` renders inline error (existing pattern) |

### Season Stats

| State | Visual |
|-------|--------|
| Loading | `ion-spinner name="crescent"` centered in card |
| Loaded with data | 3-column stat card grid |
| All zeros (no completed games) | Grid with all zeros + muted helper text below |
