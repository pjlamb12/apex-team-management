    - Inject `LiveGameStateService` and `LiveClockService`.
    - `playtimeMap` should return an object `{ [playerId: string]: number }` (total seconds).
    - Logic:
      1. Start with initial lineup (at t=0, starters have stintStart = 0).
      2. Iterate through events (filtered for non-deleted).
      3. On `SUB` (in): record stint start for player.
      4. On `SUB` (out): add `event.gameTimeMs - stintStart` to player's total, clear stint start.
      5. After events: for all currently active players (those with an open stint), add `clockService.elapsedMs() - stintStart` to their total.
      6. Convert all totals to seconds (Math.floor(ms / 1000)).
    - Ensure it handles the absence of `gameTimeMs` in older events by falling back to `minuteOccurred * 60000`.
    - Create `playtime.service.spec.ts` with test cases for:
      - Initial state (no events, clock at 0).
      - Cumulative time from multiple sub-in/sub-out events.
      - Real-time incrementing when clock is running.
      - Persistence across period ends/breaks.
  </action>
