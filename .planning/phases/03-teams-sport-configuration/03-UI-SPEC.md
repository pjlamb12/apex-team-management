## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip padding |
| sm | 8px | Input field internal gaps, list item separation |
| md | 16px | Card padding, form field stacking (mt-4), page side padding |
| lg | 24px | Card header vertical padding, section gaps |
| xl | 32px | Empty state vertical centering gap |
| 2xl | 48px | Major empty state icon/text blocks |
| 3xl | 64px | Page-level centering offset |

Exceptions:
- Touch targets for tab bar icons: 44px minimum height (Ionic `IonTabBar` handles this natively)
- FAB button: 56px diameter (Ionic `IonFab` default)

**Source:** Existing auth component patterns (mt-4, ion-padding, max-w-sm). Default Ionic spacing conventions.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Ionic `--ion-background-color` dark (`#121212` approx) | Page background, `IonContent` surface |
| Secondary (30%) | Ionic `--ion-card-background` dark (`#1e1e1e` approx) | `IonCard` backgrounds, `IonItem` backgrounds, tab bar |
| Accent (10%) | Ionic `--ion-color-primary` (default Ionic blue `#3880ff`) | Primary CTAs, active tab indicator, sport badge background, anchor/link text |
| Destructive | Ionic `--ion-color-danger` (default `#eb445a`) | Delete button in team card, AlertController "Delete Team" button |

Accent reserved for:
- [+ Create Team] FAB button or header button
- "Create Team" submit button (`IonButton` default color)
- Active tab bar tab indicator
- Sport badge chip background
- Link text: "Sign in" / "Sign up" links (carried forward from auth pages — `text-primary`)

**Source:** INFR-02 ("Athletic Professional" dark mode theme), Phase 2 D-06 (dark mode default), existing auth templates using `text-primary`, `color="danger"`, Ionic dark palette imported in styles.scss.

Dark mode is the **default** theme (OS-preference detection + toggle persists to localStorage via ThemeService). All color specifications reference the dark palette state.

## Screen Inventory

### Screen 1: Shell (Navigation Wrapper)

**Route:** Authenticated root — wraps all team routes
**Layout:** `IonTabs` with `IonTabBar` at bottom
**Tab bar:** Single tab — "Teams" with `people-outline` Ionicon
**Auth gate:** Shell route has `authGuard` applied (carries forward from Phase 2)

### Screen 2: Teams List (`/teams`)

**Layout:** `IonHeader` + `IonToolbar` + `IonContent`
**Header:** Title "My Teams" + [+ Add] `IonButton` in `IonButtons` (end slot)
**Content:** `IonList` of `IonCard` items OR empty state

**Team card anatomy:**
- Card title: team name (20px bold)
- Sport badge: `IonBadge` "Soccer" in accent color, 14px regular
- Action row: [Edit] `IonButton` (fill="clear") + [Delete Team] `IonButton` (fill="clear", `color="danger"`)

**Empty state (no teams):**
- Centered column layout: icon + heading + body + button
- Icon: `IonIcon` name `people-outline`, 64px, muted color (60% opacity)
- Heading: "No teams yet" (20px bold)
- Body: "Create your first team to get started." (14px regular, muted)
- CTA: `IonButton` expand="block" (max-w-xs centered) — "Create Team"

**Populated state (teams present) — Focal Points:**
- Primary anchor: team name (`IonCardTitle`, 20px bold) — first thing the eye lands on in each card
- Secondary: sport badge chip (`IonBadge`, accent color) — color draws attention to sport classification
- Tertiary: action row (Edit / Delete Team buttons, fill="clear") — low visual weight until needed

### Screen 3: Create Team Form (`/teams/new`)

**Layout:** `IonHeader` (back button + title "New Team") + `IonContent`
**Form fields:**
1. Team name — `IonItem` > `IonInput` label="Team Name" labelPlacement="floating" required
2. Sport — `IonItem` > `IonSelect` label="Sport" labelPlacement="floating" — pre-selected "Soccer"
**Error display:** `ControlErrorsDisplayComponent` from `ngx-reactive-forms-utils` (same pattern as auth forms)
**Submit:** `IonButton` expand="block" "Create Team" — disabled while loading, shows "Creating..." while in flight
**Post-submit:** navigate to `/teams`

### Screen 4: Team Detail / Edit (`/teams/:id`)

**Layout:** `IonHeader` (back button + title "Edit Team") + `IonContent`
**Editable fields:**
1. Team name — `IonItem` > `IonInput` label="Team Name" labelPlacement="floating"
**Read-only fields:**
- Sport: plain text label "Sport: Soccer" — `IonItem` with `IonLabel` (no input, no edit control)
**Submit:** `IonButton` expand="block" "Save Changes" — disabled while loading
**Destructive action:** [Delete Team] `IonButton` fill="clear" color="danger" below form — triggers `AlertController` dialog

## Interaction Contracts

### Create Team flow
1. Coach taps [+ Add] in header OR "Create Team" in empty state
2. Navigate to `/teams/new`
3. Coach types team name, selects sport (Soccer pre-selected)
4. Taps "Create Team" — button shows "Creating..." — disabled
5. On success: navigate back to `/teams`
6. On API error: show `IonText color="danger"` inline error — "Failed to create team. Please try again."

### Edit Team flow
1. Coach taps [Edit] on team card
2. Navigate to `/teams/:id`
3. Coach edits team name (sport is read-only)
4. Taps "Save Changes" — button shows "Saving..." — disabled
5. On success: navigate back to `/teams`
6. On API error: show `IonText color="danger"` inline — "Failed to save. Please try again."

### Delete Team flow (D-12)
1. Coach taps [Delete Team] on team card OR [Delete Team] on edit page
2. `AlertController` fires immediately — no navigation
3. Alert dialog:
   - Header: "Delete Team"
   - Message: "Are you sure you want to delete [team name]? This cannot be undone."
   - Buttons: [Keep Team] (role: "cancel") + [Delete Team] (role: "destructive", `color="danger"`)
4. On "Keep Team": dismiss alert, no action
5. On Delete confirm: call API, dismiss alert on success, remove card from list
6. On API error: dismiss alert, show toast/inline error — "Failed to delete team. Please try again."

### Loading states
- Teams list: show `IonSkeletonText` or `ion-spinner` inside card placeholders while `GET /teams` is in flight
- All form submit buttons: disabled + label changes to past-progressive ("Creating...", "Saving...")

### Sport dropdown
- Loaded from `GET /sports` on component init
- Filtered to `isEnabled: true` only
- Soccer pre-selected (first enabled sport, or matched by name)
- Disabled after team creation (read-only on edit page)

## Accessibility Notes

- All `IonInput` fields use `labelPlacement="floating"` (accessible label always visible)
- `AlertController` dialogs are natively accessible via Ionic (focus trap, ARIA role="dialog")
- Tab bar `IonTabButton` includes both icon and text label (not icon-only)
- Color is never the sole differentiator for state — destructive actions also use "danger" text label
- Touch targets: all interactive elements meet 44px minimum via Ionic's default sizing
- `IonFabButton` must include `aria-label="Create Team"` — FAB contains only an icon with no visible text label
