## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip padding |
| sm | 8px | Input field internal gaps, list item separation, segment button padding |
| md | 16px | Card padding, form field stacking (mt-4), page side padding, lineup slot row padding |
| lg | 24px | Card header vertical padding, section gaps, divider spacing between upcoming/past groups |
| xl | 32px | Empty state vertical centering gap |
| 2xl | 48px | Major empty state icon/text blocks |
| 3xl | 64px | Page-level centering offset |

Exceptions:
- Touch targets for interactive elements (IonSegment buttons, IonSelect pickers): 44px minimum height (Ionic default)
- FAB button: 56px diameter (Ionic `IonFab` default)
- Lineup position slot rows: 48px minimum height for comfortable single-handed tap on mobile

**Source:** Carried forward from Phase 3 UI-SPEC. Lineup slot rows require 48px minimum for coach use on a mobile device during game preparation.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Ionic `--ion-background-color` dark (`#121212` approx) | Page background, `IonContent` surface, lineup editor background |
| Secondary (30%) | Ionic `--ion-card-background` dark (`#1e1e1e` approx) | `IonCard` backgrounds for game cards, `IonItem` slot rows, segment tab bar |
| Accent (10%) | Ionic `--ion-color-primary` (default Ionic blue `#3880ff`) | Active IonSegment tab indicator, "New Game" FAB button, "Create Game" submit button, "Save Lineup" button |
| Destructive | Ionic `--ion-color-danger` (default `#eb445a`) | Delete Game item-option swipe action, AlertController "Delete Game" confirm button |

Accent reserved for:
- [+ New Game] FAB button on the Games segment
- "Create Game" submit button on the game creation form
- "Save Lineup" submit button on the lineup editor
- Active `IonSegment` tab indicator (the selected Roster/Games segment underline/highlight)
- Uniform color chip background when the coach chooses team primary color (if no explicit color is set, accent is NOT used — show a neutral placeholder)

Second semantic color:
- Uniform color chip: rendered using the actual uniform color string the coach inputs (e.g., "Red" maps to Tailwind `bg-red-500`). This is a data-driven display color, not a design token. Supported named values: `red`, `blue`, `green`, `yellow`, `white`, `black`, `orange`, `purple`. Fallback: neutral secondary surface color with "—" label.

Dark mode is the **default** theme (OS-preference detection + toggle persists to localStorage via ThemeService). All color specifications reference the dark palette state.

**Source:** INFR-02, Phase 3 UI-SPEC color contract, CONTEXT.md D-07 (uniformColor field). ThemeService established in Phase 1.

## Screen Inventory

### Screen 1: Team Dashboard — Games Segment (view within `/teams/:id`)

**Layout:** `IonContent` (shared with Roster segment) — `IonSegment` strip at top, content area below
**Segment strip:** Two `IonSegmentButton` — "Roster" (left) + "Games" (right). Selected Games tab shows accent indicator.
**Games content:**
- Grouped list: "Upcoming" section header + games sorted by `scheduledAt` ASC; "Past" section header + games sorted by `scheduledAt` DESC
- Each game: `IonCard` with opponent, date/time, location, uniform color chip
- FAB: `IonFab` slot="fixed" vertical="bottom" horizontal="end" — triggers navigation to `/teams/:id/games/new`

**Game card anatomy:**
- Card title: opponent name (20px bold) — "vs [Opponent]"
- Metadata row: date/time (14px regular, muted) + location (14px regular, muted)
- Bottom row: uniform color chip (`IonBadge`) + status badge "Scheduled" (14px, secondary surface)
- Action row: [Set Lineup] `IonButton` fill="outline" size="small" — navigates to `/teams/:id/games/:gameId/lineup`

**Empty state (no games):**
- Centered column layout: icon + heading + body + button
- Icon: `IonIcon` name `calendar-outline`, 64px, muted color
- Heading: "No games yet" (20px bold)
- Body: "Add your first game to start building lineups." (14px regular, muted)
- CTA: `IonButton` expand="block" max-w-xs centered — "New Game"

**Populated state focal points:**
- Primary anchor: opponent name ("vs Thunder FC") — first thing the eye lands on
- Secondary: date/time — when it's happening
- Tertiary: location, color chip, status badge — supporting context

**Section divider between Upcoming / Past:**
- `IonItemDivider` or Tailwind `border-b` with section label text "Upcoming" / "Past Games" (14px regular, muted, uppercase tracking)

### Screen 2: Create Game Form (`/teams/:id/games/new`)

**Layout:** `IonHeader` (back button + title "New Game") + `IonContent` with `ion-padding`
**Card wrapper:** `IonCard` max-w-md centered (matches Phase 3/4 form pattern)
**Form fields (in order):**
1. Opponent — `IonItem` > `IonInput` label="Opponent" labelPlacement="floating" type="text" required placeholder="e.g. Thunder FC"
2. Location — `IonItem` > `IonInput` label="Location" labelPlacement="floating" type="text" placeholder="e.g. City Park Field 2"
3. Date & Time — `IonItem` > `IonDatetimeButton` + `IonModal` > `IonDatetime` (presentation="date-time") required
4. Uniform Color — `IonItem` > `IonSelect` label="Uniform Color" labelPlacement="floating" — options: Red, Blue, Green, Yellow, White, Black, Orange, Purple

**Error display:** `ngx-control-errors-display` (`ControlErrorsDisplayComponent`) wrapping each `IonItem` — same pattern as all prior form pages
**Submit:** `IonButton` expand="block" class="mt-4" "Create Game" — disabled while saving, shows "Creating..."
**Post-submit:** Navigate directly to `/teams/:id/games/:gameId/lineup` (CONTEXT.md Claude's Discretion — immediate lineup entry)

### Screen 3: Game Edit (`/teams/:id/games/:gameId/edit`)

**Layout:** `IonHeader` (back button + title "Edit Game") + `IonContent`
**Fields:** Same 4 fields as Create Game, pre-populated with existing values
**Submit:** `IonButton` expand="block" "Save Changes" — disabled while saving, shows "Saving..."
**Destructive action:** [Delete Game] `IonButton` fill="clear" color="danger" below form — triggers `AlertController` dialog
**Post-submit:** Navigate back to `/teams/:id` (Games segment visible)

### Screen 4: Lineup Editor (`/teams/:id/games/:gameId/lineup`)

**Layout:** `IonHeader` (back button + title "Starting Lineup" subtitle "[Opponent] — [Date]") + `IonContent`

**On-Field section (11 slots):**
- Section header: "On Field" (14px regular, uppercase, muted) — `IonItemDivider` or styled `div`
- 11 `IonItem` rows, each containing:
  - Left: slot number label (14px regular, muted) — e.g. "1", "2" ... "11"
  - Center-left: Position picker — `IonSelect` label="Position" (inline, no floating label) — options: GK, DEF, MID, FWD (from sport's positionTypes)
  - Center-right: Player picker — `IonSelect` label="Player" (inline, no floating label) — options: available roster players shown as "#[jersey] [First] [Last]". Players already assigned to another slot are excluded.
  - Row height: 48px minimum
- Each slot row uses `IonItem` with `lines="full"` — clear visual separation between slots

**Bench section:**
- Section header: "Bench" (14px regular, uppercase, muted) — `IonItemDivider`
- `IonList` of bench players (roster players not assigned to any on-field slot)
- Each bench player: `IonItem` — jersey badge (`IonBadge` color="medium") + "[First] [Last]" (14px regular)
- Bench list is read-only — players are auto-populated as "not assigned to a slot"
- Empty bench state: `IonItem` > `IonLabel` — "All players assigned to on-field slots." (14px regular, muted)

**Save button:**
- `IonButton` expand="block" class="mt-4 mx-4 mb-4" "Save Lineup" — accent color, disabled while saving, shows "Saving..."
- Positioned at bottom of content (not FAB — this is a form submit)

**Validation:**
- If a player slot has a position but no player (or vice versa), show `IonNote` slot="error" on the item — "Select both a position and a player."
- Duplicate player detection is prevented via picker filtering (excluded from other selects), not a separate validation message

**Post-save:** Navigate back to `/teams/:id` (Games segment, game card now shows [Set Lineup] button replaced by [Edit Lineup] if lineup exists)

## Interaction Contracts

### IonSegment tab switch (Roster ↔ Games)
1. Coach taps "Games" segment button on Team Dashboard
2. Segment value changes → component's `selectedSegment` signal updates
3. Roster list hides; Games list becomes visible (Angular `@if` block switch)
4. Games list loads from signal (populated if already loaded, triggers API call if empty)
5. No navigation — inline content swap within same route

### New Game flow
1. Coach taps [+ New Game] FAB on Games segment
2. Navigate to `/teams/:id/games/new`
3. Coach fills: Opponent (required), Location (optional), Date & Time (required), Uniform Color (optional)
4. Taps "Create Game" — button shows "Creating..." — disabled
5. On success: navigate to `/teams/:id/games/:gameId/lineup`
6. On API error: show `IonText color="danger"` inline — "Failed to create game. Please try again."

### Set / Edit Lineup flow
1. Coach taps [Set Lineup] or [Edit Lineup] on a game card OR arrives via post-game-creation redirect
2. Navigate to `/teams/:id/games/:gameId/lineup`
3. Coach selects position type and player for each of the 11 on-field slots
4. Bench section auto-updates as players are assigned (removes them from bench display)
5. Taps "Save Lineup" — button shows "Saving..." — disabled
6. On success: navigate back to `/teams/:id` (Games segment active)
7. On API error: show `IonText color="danger"` inline — "Failed to save lineup. Please try again."

### Edit Game flow
1. Coach taps [Edit] on game card (swipe reveal) OR header edit button
2. Navigate to `/teams/:id/games/:gameId/edit`
3. Coach edits any fields
4. Taps "Save Changes" — button shows "Saving..." — disabled
5. On success: navigate back to `/teams/:id`
6. On API error: show `IonText color="danger"` inline — "Failed to save. Please try again."

### Delete Game flow
1. Coach swipes game card left → `IonItemSliding` reveals trash icon `IonItemOption color="danger"`
2. Coach taps trash → `AlertController` fires immediately
3. Alert dialog:
   - Header: "Delete Game"
   - Message: "Are you sure you want to delete the game vs [Opponent]? This cannot be undone."
   - Buttons: [Keep Game] (role: "cancel") + [Delete Game] (role: "destructive", `color="danger"`)
4. On "Keep Game": dismiss alert, no action, slide resets
5. On Delete confirm: call API, dismiss alert, remove card from list
6. On API error: dismiss alert, show `IonText color="danger"` inline — "Failed to delete game. Please try again."

### Position picker interaction
1. Coach taps position `IonSelect` on a lineup slot
2. Ionic action sheet or alert opens with options: GK, DEF, MID, FWD
3. Coach selects a position type — picker closes, slot updates
4. No save triggered — selection is staged until "Save Lineup" tapped

### Player picker interaction
1. Coach taps player `IonSelect` on a lineup slot
2. Ionic action sheet opens with available players — format "#[jersey] [First] [Last]"
3. Players already assigned to other slots are excluded from the list
4. Coach selects a player — picker closes, slot updates, player disappears from bench section
5. No save triggered — selection is staged until "Save Lineup" tapped

### Loading states
- Games list: `IonSpinner` name="crescent" centered while `GET /teams/:id/games` is in flight
- Lineup editor: `IonSpinner` centered while `GET /games/:gameId/lineup` and `GET /teams/:id/players` are loading
- All form submit buttons: disabled + label changes to past-progressive ("Creating...", "Saving...")

## Accessibility Notes

- All `IonInput` fields use `labelPlacement="floating"` (accessible label always visible when field is populated)
- `IonSelect` components use the `label` prop — label is always associated with the select control
- `AlertController` dialogs are natively accessible via Ionic (focus trap, ARIA role="dialog")
- `IonSegment` is keyboard-navigable via Ionic's native implementation
- Lineup slot rows at 48px minimum height satisfy touch target requirements on mobile
- Color is never the sole differentiator for state — uniform color chips also show a text label ("Red", "Blue", etc.)
- `IonFabButton` must include `aria-label="New Game"` — FAB contains only an icon
- Swipe-to-delete action (`IonItemSliding`) is supplemented by the Edit Game page which also offers delete — swipe gesture is not the only access path
- `IonSpinner` loading states should include an `aria-label="Loading..."` wrapper for screen readers
