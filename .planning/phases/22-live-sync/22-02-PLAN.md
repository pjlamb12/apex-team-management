# Plan 22-02: Frontend Real-time State Updates

Update the frontend to listen for Socket.io events and update the local state accordingly.

## Tasks

### 1. Update LiveGameStateService to handle remote events
- Add methods to `LiveGameStateService` to merge remote events into the local state.
- Handle `gameEventLogged`: if the event ID is already present, update it; otherwise, append it.
- Handle `gameEventRemoved`: remove the event from the state.

### 2. Update ConsoleWrapper to listen for events
- In `ngOnInit` (or where the socket joins the room), subscribe to `gameEventLogged`, `gameEventRemoved`, and `gameStatusUpdated`.
- When an event is received, call the appropriate method in `LiveGameStateService`.

### 3. Handle Status Synchronization
- Update the UI when `gameStatusUpdated` is received (e.g., refresh event data).

## Verification Strategy

### Manual Verification
- Log in as two different users (e.g., in Chrome and Firefox).
- Open the same game in both.
- Add a goal as User A and verify it appears instantly for User B.
- Make a substitution as User A and verify User B's lineup updates instantly.
