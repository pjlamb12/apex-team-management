# Plan 22-01: Backend Event Broadcasting

Update the backend to broadcast events to the relevant Socket.io rooms whenever state changes occur.

## Tasks

### 1. Inject SocketGateway into EventsService
- Update `EventsService` to depend on `SocketGateway`.

### 2. Emit Events on Status Changes
- In `update` (and specifically when status or currentPeriod changes), emit a `gameStatusUpdated` event to the `event:{eventId}` room.

### 3. Emit Events on Game Logs
- In `logEvent`, emit a `gameEventLogged` event to the `event:{eventId}` room.
- In `removeEvent`, emit a `gameEventRemoved` event to the room.

### 4. Emit Events on Event Creation
- In `create`, emit an `eventCreated` event to the `team:{teamId}` room.

## Verification Strategy

### Automated Tests
- Update `EventsService` unit tests to verify that `SocketGateway` methods are called with the correct arguments.

### Manual Verification
- Use two browser windows (or a browser and Postman) to verify that an event emitted by one window is received by the other.
