  <action>
Ensure both the API and frontend are running before verification.

**Start services (in separate terminals or via nx run-many):**
```bash
# Terminal 1 — start PostgreSQL if not running
docker compose up -d postgres

# Terminal 2 — start NestJS API
nx run api:serve

# Terminal 3 — start Angular frontend
nx run frontend:serve
```

Wait until both servers are ready:
- API: `curl -s http://localhost:3000/api/health` returns `{"status":"ok"}`
- Frontend: `http://localhost:4200` loads in browser

**API endpoint smoke tests (run sequentially):**

Test 1 — Signup (AUTH-01):
```bash
curl -s -X POST http://localhost:3000/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"coach@apex.test","password":"TestPass123","displayName":"Test Coach"}' \
  | jq '{hasToken: (.accessToken != null and .accessToken != "")}'
```
Expected: `{ "hasToken": true }`

Test 2 — Duplicate email returns 409:
```bash
curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:3000/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"coach@apex.test","password":"TestPass123","displayName":"Test Coach"}'
```
Expected: `409`

Test 3 — Login (AUTH-02):
```bash
TOKEN=$(curl -s -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"coach@apex.test","password":"TestPass123"}' \
  | jq -r '.accessToken')
echo "Token length: ${#TOKEN}"
```
Expected: Token length > 50

Test 4 — Invalid credentials return 401:
```bash
curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"coach@apex.test","password":"WrongPass"}'
```
Expected: `401`

Test 5 — Refresh (AUTH-02 sliding window):
```bash
NEW_TOKEN=$(curl -s -X POST http://localhost:3000/api/auth/refresh \
  -H "Authorization: Bearer $TOKEN" \
  | jq -r '.accessToken')
echo "Refresh token length: ${#NEW_TOKEN}"
```
Expected: New token length > 50

Test 6 — Forgot-password (AUTH-04):
```bash
curl -s -X POST http://localhost:3000/api/auth/forgot-password \
  -H "Content-Type: application/json" \
  -d '{"email":"coach@apex.test"}' | jq .message
```
Expected: `"If that email exists, a reset link has been sent."` AND a `[PasswordResetService] Reset URL:` line visible in the NestJS console.

Test 7 — Reset-password (AUTH-04):
Copy the token from the NestJS console log in Test 6, then:
```bash
curl -s -X POST http://localhost:3000/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d "{\"token\":\"<PASTE_TOKEN_HERE>\",\"newPassword\":\"NewPass456\"}" | jq .message
```
Expected: `"Password reset successful"`

Test 8 — Login with new password:
```bash
curl -s -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"coach@apex.test","password":"NewPass456"}' | jq '{hasToken: (.accessToken != null)}'
```
Expected: `{ "hasToken": true }`

Test 9 — DTO validation returns 400:
```bash
curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:3000/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"not-an-email","password":"short"}'
```
Expected: `400`
