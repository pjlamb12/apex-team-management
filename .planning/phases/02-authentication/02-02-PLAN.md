**Do NOT run `nx run api:migration:generate`** for this migration — the generator requires the database to diff against and may produce a different timestamp. Instead, write the migration file manually using the exact timestamp `1744990000000` so the filename is deterministic and matches the `depends_on` reference in 02-03-PLAN.md.

Create `apps/api/src/migrations/1744990000000-AddAuthColumns.ts` with the following content:

```typescript
import { MigrationInterface, QueryRunner } from 'typeorm';

export class AddAuthColumns1744990000000 implements MigrationInterface {
  name = 'AddAuthColumns1744990000000';

  public async up(queryRunner: QueryRunner): Promise<void> {
    // Two-step NOT NULL column addition: add with DEFAULT '' first,
    // then DROP DEFAULT so existing rows get the placeholder value
    // and new rows must supply a real hash. (RESEARCH.md migration pitfall)
    await queryRunner.query(`
      ALTER TABLE "users"
        ADD COLUMN "password_hash" character varying NOT NULL DEFAULT '',
        ADD COLUMN "password_reset_token" character varying,
        ADD COLUMN "password_reset_expiry" TIMESTAMP
    `);
    await queryRunner.query(`
      ALTER TABLE "users" ALTER COLUMN "password_hash" DROP DEFAULT
    `);
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`
      ALTER TABLE "users"
        DROP COLUMN IF EXISTS "password_hash",
        DROP COLUMN IF EXISTS "password_reset_token",
        DROP COLUMN IF EXISTS "password_reset_expiry"
    `);
  }
}
```

After writing the file, run the migration against the running PostgreSQL container:

```bash
nx run api:migration:run
```

The command uses `TS_NODE_PROJECT=apps/api/tsconfig.app.json npx typeorm-ts-node-commonjs migration:run -d apps/api/src/data-source.ts` internally (per `apps/api/project.json`).

If the migration fails because PostgreSQL is not running, start the container first:
```bash
docker compose up -d postgres
```
Then retry `nx run api:migration:run`.

After the migration succeeds, verify the columns exist in the database:
```bash
docker exec -it apex-team-postgres-1 psql -U postgres -d apex_team -c "\d users"
```
(Container name may vary; use `docker ps` to find the exact name if needed.)
