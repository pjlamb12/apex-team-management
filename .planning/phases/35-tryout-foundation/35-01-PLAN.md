# Phase 35: Tryout Foundation

## Goal
Establish the core data infrastructure for managing prospective players (Candidates) and tryout-specific events.

## Requirements
- **TRY-CAND-01**: Create a `Candidate` entity to store contact information and basic player details.
- **TRY-CAND-02**: Extend the `Event` entity to support a "tryout" type.
- **TRY-CAND-03**: Create a `CandidateAttendance` entity to track candidate presence at tryout sessions.

## Proposed Data Schema

### CandidateEntity
- `id`: UUID (Primary Key)
- `firstName`: string
- `lastName`: string
- `dateOfBirth`: date (nullable)
- `parentName`: string (nullable)
- `parentEmail`: string
- `parentPhone`: string (nullable)
- `preferredPosition`: string (nullable)
- `notes`: text (nullable)
- `status`: enum ('interested', 'invited', 'offered', 'accepted', 'declined')
- `teamId`: string (FK to TeamEntity)

### EventEntity (Updates)
- Update `type` column to allow `'tryout'`: `'game' | 'practice' | 'tryout'`

### CandidateAttendanceEntity
- `id`: UUID (Primary Key)
- `candidateId`: string (FK to CandidateEntity)
- `eventId`: string (FK to EventEntity)
- `status`: enum ('present', 'absent', 'tardy')

3. Candidate attendance can be recorded for tryout events.
