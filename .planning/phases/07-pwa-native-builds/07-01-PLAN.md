# Plan 07-01: PWA and Capacitor Initialization

**Phase:** 07-pwa-native-builds
**Plan:** 01
**Goal:** Initialize the Progressive Web App (PWA) manifest and service worker, then set up the Capacitor foundation for native builds.

## Requirements
- **INFR-04:** App is installable as a PWA with manifest and service worker.
- **INFR-05:** Capacitor configured for Android.
- **INFR-06:** Capacitor configured for iOS.

## Context
- Angular 21 project in `apps/frontend/`.
- `@ionic/angular` already installed (Phase 1).
- `@angular/service-worker` already present in `package-lock.json` but not configured.
- No `capacitor.config.ts` exists.

## Tasks
1. **Initialize PWA:** Use `@angular/pwa` to generate the manifest, service worker config, and icons.
2. **Setup Capacitor:** Install Capacitor core/cli and initialize the project.
3. **Configure Web Assets:** Ensure Capacitor points to the correct build output (`dist/apps/frontend/browser`).
4. **Initial Verification:** Confirm the manifest is served and `npx cap` commands are available.

## Implementation Strategy
- `npx ng add @angular/pwa --project=frontend`: This will update `package.json`, create `src/manifest.webmanifest`, `src/ngsw-config.json`, and add the service worker registration to `app.config.ts`.
- `npm install @capacitor/core @capacitor/cli @capacitor/android @capacitor/ios`: Install Capacitor dependencies.
- `npx cap init "Apex Team" "com.apexteam.app" --web-dir "dist/apps/frontend/browser"`: Initialize Capacitor with the current app name and build directory.

## Verification Plan
- **PWA Check:** Build the app (`npx nx build frontend`) and verify `manifest.webmanifest` and `ngsw.json` exist in the output.
- **Capacitor Check:** Run `npx cap sync` and verify it detects the build output.
