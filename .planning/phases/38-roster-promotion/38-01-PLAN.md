# Phase 38: Roster Promotion

## Goal
Enable coaches to convert successful tryout candidates into active roster players.

## Requirements
- **TRY-PROM-01**: Implement a backend endpoint to "promote" a candidate to a player.
- **TRY-PROM-02**: Prevent duplicate player creation if a candidate is already on the roster.
- **TRY-PROM-03**: Build a "Promotion" UI in the Tryout Console to select candidates for onboarding.

## Implementation Tasks

### 1. Backend (NestJS)
- [ ] Add `promoteCandidate` method to `CandidatesService`.
- [ ] Add `POST /teams/:teamId/candidates/:id/promote` endpoint to `CandidatesController`.
- [ ] Ensure `Candidate` status is updated to `'accepted'` upon successful promotion.

### 2. Frontend (Angular)
- [ ] Add `promoteCandidate` to `CandidatesService` in data-access library.
- [ ] Build a "Promotion" tab or modal in the Tryout Console.
- [ ] Add visual feedback for already-promoted players.

## Success Criteria
1. Candidates can be converted to `Player` entities with one click.
2. Promoted players appear in the main Team Roster immediately.
3. Candidate status updates to reflects their new roster position.
